#4  Tracepoint whitelist — drop unused ros2_tracing events before rerun
    Pushed: 2026-04-20 22:05 CEST

    Context: 36.5M ros2_trace events on the Autoware 150 s rosbag
    replay (fish_20260419_211621), most of which FISH never reads.
    Consequences:

      - LTTng buffer overflow: 121,632 events discarded (0.76%) —
        notes/known_issues.txt + immediate_work.txt #0d
      - Ingest time dominated by bt2 streaming of events we throw
        away (~17 min on the Autoware run)
      - rclpy timer re-init inflation (ISSUE-001): 88K+ useless
        rcl_timer_init events on a 30 s session

    Approach:
      - Inventory what FISH actually consumes from ros2_trace —
        search postprocess/ for `event`:`ros2:*` queries and list
        unique event names. Current inventory (from memory):
          ros2:callback_start, ros2:callback_end, ros2:rcl_publish,
          ros2:fish_rclcpp_client_request_sent,
          ros2:rcl_node_init, ros2:rcl_subscription_init,
          ros2:rcl_publisher_init, ros2:rcl_client_init,
          ros2:rcl_service_init, ros2:rcl_timer_init,
          ros2:rclcpp_subscription_callback_added,
          ros2:rclcpp_service_callback_added,
          ros2:rclcpp_client_callback_added,
          ros2:rclcpp_timer_callback_added,
          ros2:rclcpp_subscription_init,
          ros2:fish_*  (our own set — all needed)
          ros2:rclpy_*  (our rclpy parity — all needed)
      - Compile LTTng enable-event whitelist (`-u 'ros2:callback_start'
        -u ...`) to replace current wildcard enable
      - Audit: what would we LOSE? Anything in ingest.py loops that
        isn't in the whitelist needs to stay or needs an explicit
        decision (accept loss).
      - Apply to trace_session.sh (or wherever `lttng enable-event`
        is issued)
      - Bump LTTng subbuf-size/num-subbuf while we're there — smaller
        event volume means less risk of overflow even without buffer
        tuning, but bigger buffers are cheap safety.

    Acceptance:
      - Tracepoint count inventory in notes/tracepoint_inventory.txt
      - trace_session.sh uses explicit enable-event whitelist
      - Rerun (item #2) shows ros2_trace row count materially lower
        (target: ~3-5x reduction), ingest time down proportionally,
        LTTng discards = 0
      - Model extraction + gpu_task_graph identical structural
        output vs pre-whitelist baseline (no signal lost)

    Sequencing: do this BEFORE item #2 so the rerun captures both
    the nsys flag changes AND the trimmed tracepoint set in one
    pass. Otherwise we'd need two reruns.

    Next action after pop: grep postprocess/ for "ros2:" event
    consumers and draft the whitelist.
