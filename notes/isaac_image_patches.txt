Isaac ROS benchmark image patches required for FISH integration
================================================================

When shipping the FISH-bundled fish-r2b-<bench> images, the following
modifications MUST be baked in (currently applied at sweep runtime by
overnight_sweep_v3.sh — works for measurement but not portable).

Last updated: 2026-06-02 — adds patches [1a]+[10..16] discovered during the
second overnight retry sweep (foundationpose / detectnet / centerpose /
ess / segformer / unet family).

[1] Install TensorRT binary (libnvinfer-bin)
---------------------------------------------

  Symptom (before fix):
    launch_test: error: [Errno 2] No such file or directory:
      '/usr/src/tensorrt/bin/trtexec'

  Affected benchmarks: bi3d, bi3d_freespace, foundationpose, centerpose,
                       rtdetr, dope, segment_anything (7 of 23).

  Root cause: fish-r2b-* images ship libnvinfer-dev (headers + dev libs)
  but not libnvinfer-bin (which provides the trtexec compiler binary at
  /usr/src/tensorrt/bin/trtexec). Isaac ROS benchmark Python scripts
  invoke trtexec directly to convert ONNX models to TensorRT engine
  plans at launch-time.

  Fix (one-line apt):
    RUN apt-get install -y libnvinfer-bin
  Verified: matches existing libnvinfer-dev 10.16.1.11-1+cuda13.2.

  Currently applied: sweep_v3 runs the apt-install inside each
  container's bash -lc preamble.


[2] Symlink /workspaces/isaac_ros-dev -> /root/ros_ws
-----------------------------------------------------

  Symptom (before fix):
    Input file cannot be found: /workspaces/isaac_ros-dev/src/
      ros2_benchmark/assets/models/bi3d/featnet.onnx

  Affected: every benchmark that calls TRTConverter — same 7 as [1]
  plus any benchmark using assets/models/.

  Root cause: NVIDIA's isaac_ros-dev container template uses
  /workspaces/isaac_ros-dev as the workspace root. Our fish-r2b-*
  images put it at /root/ros_ws. Some Isaac ROS Python scripts have
  the workspaces/ path hardcoded (not parameterised via env).

  Fix (Dockerfile):
    RUN mkdir -p /workspaces && ln -sfn /root/ros_ws /workspaces/isaac_ros-dev

  Currently applied: sweep_v3 creates the symlink at container start.

[4] FISH install from host source at container start
-----------------------------------------------------

  Currently each sweep_v3 run does:
    - rm -rf /root/fish_interfere    (drops stale baked-in copy)
    - cp -rT /host/fish_src /root/fish_interfere
    - bash setup_fish.sh --yes

  This ~30 s install happens per container so the FISH wrapper +
  Python modules always match the host's current code (helpful during
  development).

  For SHIPPED images: bake FISH into the image instead via the existing
  container_install.sh flow. Once stable, run install_fish.sh once and
  docker commit. Then the runtime cp+install steps can be skipped.

[1a] libnvinfer-bin install needs `apt-get update` + version pin
----------------------------------------------------------------

  Refines [1]. Most fish-r2b-* images ship with stale apt cache.

  Symptom (before fix):
    bi3d_freespace, foundationpose, centerpose, rtdetr, dope,
    segment_anything ALL failed with:
      E: Unable to locate package libnvinfer-bin
    (even though [1] said "just apt install libnvinfer-bin").

  Root cause:
    1. apt cache is stale — libnvinfer-bin not listed without `apt-get update`.
    2. Unversioned install resolves to latest (11.x), but images carry
       libnvinfer-dev 10.16.1.11 → dep mismatch (needs libnvinfer11).

  Fix (in sweep_v3.sh preamble — see lines ~221-233):
    if [ ! -x /usr/src/tensorrt/bin/trtexec ]; then
        NVINFER_VER=$(dpkg-query -W -f='${Version}' libnvinfer-dev 2>/dev/null)
        apt-get update >/tmp/apt_update.log 2>&1 || true
        apt-get install -y "libnvinfer-bin=$NVINFER_VER"
    fi

  Currently applied: sweep_v3 runtime.
  For shipped image: bake libnvinfer-bin into the image at build time.

  works inside the image. It does NOT, because:
    - The image carries `isaac_ros_*_models_install` packages with
      COLCON_IGNORE files (excluded from default build).
    - Even when built, the ROS install path differs from where the
      benchmark expects the ONNX/config.pbtxt.
    - Some benchmarks expect a DIFFERENT model than the install_script
      provides (e.g., detectnet expects resnet34_peoplenet (TAO) but
      install_peoplenet_amr_rs.sh installs rsu_rs_480_640_mask (Isaac AMR)).

  Solution: pre-download all needed ONNX from NGC into manual_models/,
  copy them into the benchmark's expected path at sweep runtime.

[12] r2bdataset2024 — r2b_robotarm bag
---------------------------------------

  Symptom: foundationpose, rtdetr, segment_anything all fail with
    RuntimeError: Bag path .../r2b_dataset/r2b_robotarm does not exist.

  Root cause: r2b_robotarm is in r2bdataset2024 (released later), not
  the r2bdataset2023_v3 we were mounting.

  Fix:
    1. User downloads: ngc registry resource download-version
                       "nvidia/isaac/r2bdataset2024:1" → ~/r2bdataset2024_v1/
    2. sweep_v3.sh mounts BOTH datasets and symlinks all bags:
         -v "$R2B_DATASET":/host/r2bdataset:ro              (2023)
         -v "$R2B_DATASET_2024":/host/r2bdataset_2024:ro    (2024)
       Then inside the container preamble:
         for bag in /host/r2bdataset/* /host/r2bdataset_2024/*; do
             ln -sf "$bag" "$DEST_DS/$(basename "$bag")"
         done

  For shipped image: this is per-session data — keep at runtime mount.


[13] heredoc variable escaping in docker bash -lc
-------------------------------------------------

  Bug class — not strictly an image patch, but breaks runs.

  Symptom:
    scripts/overnight_sweep_v3.sh: line 184: NVINFER_VER: unbound variable
    → sweep dies before any benchmark runs.

  Root cause: docker bash -lc heredoc uses double-quotes — outer shell
  with `set -u` expands `$NVINFER_VER` and aborts because the variable
  isn't defined in the OUTER scope.

  Fix: escape every shell variable that must resolve INSIDE the container:
    \$NVINFER_VER, \${Version}, \"\$NVINFER_VER\"
  Apply to ALL new lines added to the docker heredoc.


[14] FISH watchdog must use container-side clock
-------------------------------------------------

  Symptom:
    [watchdog 01:00:28] killing stuck container peaceful_hawking
       (nsys done, 2899 s idle)
    → freshly-started detectnet container killed within 15 seconds
      because the watchdog thought its events log was "idle for 48 min".

  Root cause: /tmp/sweep_watchdog.sh compared `stat -c %Y` (container
  filesystem) with host's `date +%s`. Isaac ROS containers boot with a
  clock offset (often ~10 h skewed from host).

  Fix (already applied to /tmp/sweep_watchdog.sh):
    AGE=$(docker exec "$c" stat -c %Y "$EVT")
    NOW=$(docker exec "$c" date +%s)        # ← container-side clock
    DELTA=$((NOW - AGE))


[15] sweep_postprocess.sh session selector
-------------------------------------------

  Bug class — postprocess pipeline.

  Symptom: every newly-working benchmark produced fish_graph.json
  with V=1 E=0 (empty model) even though raw trace data was OK.

  Root cause: `ls -d .../fish_* | head -1` returns alphabetically-first
  match (the OLDEST session) AND, after first postprocess run, also
  matches the `fish_viz.html` and `fish_graph.json` siblings.

  Fix (in /tmp/sweep_postprocess.sh):
    SD=$(ls -td -- /tmp/fish_traces_v3/$BENCH/measure/fish_*/ \
         | head -1 | sed 's:/$::')

  `-t` → newest first; trailing `/` filters to dirs only; sed strips slash.
