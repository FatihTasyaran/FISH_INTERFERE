Paper-important observations (paper_important_points.txt P1-P6)
---------------------------------------------------------------

P1: Modern CUDA/TRT/ORT workloads are GPU-DAG-native
    (pointcloud: 3776 syncs 100 % StreamWaitEvent, yolo: 3938 type-2
    + 1 ContextSync only). CPU rarely issues blocking stream sync
    → the scheduling object is in the GPU hardware trace directly.

P2: Stream-ID disambiguator is REQUIRED under multi-threaded
    executors (32-way concurrent callback overlap observed).

P3: Callback boundary ≠ task boundary — 73 % of Autoware callbacks
    are "launch-last" (dispatch then return). Task spans multiple
    callbacks via shared stream.

P4: TRT pattern — one stream = one inference engine = one task.
    Pointcloud pid=3211 uses exactly 3 streams for all GPU work.

P5: INIT is not a time prefix — must be score-based per-period
    (init signals bleed into frame-1 on yolo).

P6: Variation shapes encode branch structure — don't discard as
    UNKNOWN. Embed with edit-distance + cluster to surface the
    driver-level kernel substitution pattern.
