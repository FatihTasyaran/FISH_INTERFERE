▼

sync-boundary pattern: cudaStreamSynchronize followed by > 5 ms idle gap marks boundary. Alternative (v2): GPU-side gap-based partition per stream.

▼
Typed-DAG builder: for each period, emit CPU/SM/CE nodes + launch, stream_fifo, cpu_order edges.
▼

▼

signature hash over (pe, name) sequence + sorted edges. Exact match only. Future: edit-distance clustering (embed_start.txt).
▼
{signature → count + template DAG}
▼
3. Phase detection — "What lifecycle role does each period play?"
init_score weighted by CUDA-init API tokens + rule classifier. Labels: INIT / WARMUP / STEADY / TAIL / COALESCED(N) / UNKNOWN.
▼
labelled periods + primary phase per shape
▼

correlationId → streamId join. Ownership rules:

Phase detection algorithm (G5)

FieldDefinition
Inputa set of periods P = { (start_i, end_i, DAG_i, api_names_i) }
Outputfor each period: label ∈ {INIT, WARMUP, STEADY, TAIL, COALESCED(N), UNKNOWN}
Sourcenotes/phase_detector.txt (G5 spec)

Step 1 — Initialization score per period

function init_score(api_names):
    # Count init-signature tokens in this period's API call list
    counts ← { }
    for name in api_names:
        for (token, threshold, weight) in INIT_SIGNALS:
            if token in name:
                counts[token] ← counts.get(token, 0) + 1

    score ← 0
    for (token, threshold, weight) in INIT_SIGNALS:
        c ← counts.get(token, 0)
        if threshold = NONE:
            score ← score + weight · c          # per-count weak signal
        else if c ≥ threshold:
            score ← score + weight              # threshold-gated hard signal
    return score

INIT_SIGNALS:
    ("cuInit",                       1,  100)     # one-time process startup
    ("cuLibraryLoadData",            1,  100)
    ("cuDevicePrimaryCtxRetain",     1,  100)
    ("cuCtxCreate",                  1,  100)
    ("cudaGetDeviceProperties",      1,   20)
    ("cuModuleGetLoadingMode",       1,   20)
    ("cuGetProcAddress",           100,   50)     # big symbol-resolution burst
    ("cudaMalloc",                NONE,    5)     # per-count weak signal
    ("cudaStreamCreate",          NONE,    3)
    ("cudaEventCreate",           NONE,    1)

Step 2 — Dominant signature over the productive subset

productive ← { p ∈ P : init_score(p) < 20 }
sig_of(p)  ← SHA1( (pe, name) tuple sequence(DAG_p) ⊕ sorted edges(DAG_p) )

dom_sig    ← argmaxs  | { p ∈ productive : sig_of(p) = s } |
dom_seq    ← node sequence of the canonical DAG with signature dom_sig

Step 3 — Classification rules (priority order)

for each period p in P:
    s    ← init_score(p)
    seq  ← node_seq(DAG_p)
    pos  ← chronological index of p in P

    # Rule 1 — hard init
    if s ≥ 100:
        label(p) ← INIT
        continue

    # Rule 2 — dominant-shape steady state
    if sig_of(p) = dom_sig:
        label(p) ← STEADY
        continue

    # Rule 3 — coalesced repeat (detector miss)
    if  |seq| mod |dom_seq| = 0
        and |seq| > |dom_seq|
        and ∀ k ∈ [0, |seq|) : seq[k] = dom_seq[k mod |dom_seq|]:
        N ← |seq| / |dom_seq|
        label(p) ← COALESCED(N)
        continue

    # Rule 4 — tail (fuzzy prefix of dominant with trailing sync-only cleanup)
    if  pos ≥ |P| − 3
        and |seq| ≥ 10
        and |seq| < |dom_seq|:
        m ← longest k such that seq[0..k] = dom_seq[0..k]
        if  m / |seq| ≥ 0.95
            and ∀ k ∈ [m, |seq|) : "Synchronize" ∈ seq[k].name:
            label(p) ← TAIL
            continue

    # Rule 5 — warmup
    if  20 ≤ s < 100
        and pos < 3:
        label(p) ← WARMUP
        continue

    # Rule 6 — fallthrough
    label(p) ← UNKNOWN
