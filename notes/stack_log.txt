2026-04-20 22:02 CEST  POP  #3  Review + apply additional nsys config flags
  Applied 2 of the 5 candidate flags; dropped 3 after discussion:

    Applied:
      - --cuda-graph-trace=node
          Preserves per-kernel records under cudaGraphLaunch.
          Required for FISH Typed DAG work; safe to enable
          unconditionally.
      - --cuda-trace-all-apis=true
          Full CPU-side CUDA API timeline. Slight overhead, no
          stability concerns.

    Dropped:
      - --trace=...,cudnn,cublas
          kernel_short_name already distinguishes cuDNN/cuBLAS
          kernels in practice; library-level attribution not
          needed for the paper task model.
      - --cpuctxsw=process-tree
          LTTng sched_switch already captures OS context switches;
          adding nsys equivalent would be redundant data and risks
          destabilising multi-threaded composable containers (same
          risk class as the flags we already narrowed for stability
          on component_container_mt).
      - --backtrace=dwarf
          Only need top-frame kernel symbols from CUPTI_KERNEL
          .shortName; deep call-graph not paper-critical.

  Files edited:
    fish_settings.ini            (keys: cuda_graph_trace,
                                  cuda_trace_all_apis + rationale
                                  block for the rejected flags)
    python/fish/gpu.py           (build_nsys_command reads the 2
                                  new settings)
    python/fish/launch_wrap.py   (NSYS_FLAGS list extended with
                                  the 2 new flags)
